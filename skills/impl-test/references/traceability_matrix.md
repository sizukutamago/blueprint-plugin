## トレーサビリティ概要

### 目的

本ドキュメントは、要件（FR/NFR）から設計成果物（ENT/API/SC）を経てテストケースまでの
追跡可能性を確保し、以下を実現する:

- 全要件がテストで検証されていることの証明
- 要件変更時の影響範囲の迅速な特定
- テスト漏れ・設計漏れの早期発見
- リリース判定時の品質エビデンス

### 管理方針

| 項目 | 方針 |
|------|------|
| 更新タイミング | 設計変更時・テストケース追加時に随時更新 |
| 管理責任者 | テストリード |
| レビュー頻度 | 各テストフェーズ開始前にレビュー |
| カバレッジ目標 | FR: 100%, NFR: 100%（測定可能な項目） |

### ID 体系

| プレフィックス | 対象 | 例 |
|---------------|------|-----|
| FR | 機能要件 | FR-001 |
| NFR | 非機能要件 | NFR-PERF-001 |
| ENT | エンティティ | ENT-User |
| API | API エンドポイント | API-001 |
| SC | 成功基準（受入基準） | SC-001 |
| TC | テストケース（機能） | TC-001 |
| PF | パフォーマンステスト | PF-001 |
| SEC | セキュリティテスト | SEC-001 |
| ACC | アクセシビリティテスト | ACC-001 |
