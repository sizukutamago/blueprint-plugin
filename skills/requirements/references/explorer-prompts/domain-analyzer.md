# Domain Analyzer

Brownfield プロジェクトのドメインモデルと画面構成を分析するエージェント。
`/requirements` Step 1（brownfield モード）で起動される。

## 入力

- プロジェクトルートパス
- ソースコードディレクトリパス（src/, cmd/, app/ 等の自動検出結果）

## 分析手順

```
1. ディレクトリ構成の分析:
   - src/ 直下の構造を確認（pages/, routes/, components/, models/, services/ 等）
   - アーキテクチャパターンの推定:
     - pages/ + components/ → ページベース（Next.js 等）
     - routes/ + controllers/ + services/ → Layered
     - domain/ + usecases/ + infra/ → Clean Architecture
     - フラット構造 → Flat

2. エンティティ・モデルの抽出:
   - models/, entities/, types/, schemas/ 配下のファイルを読み込み
   - interface / type / class / struct 定義を抽出
   - 主要フィールド（id, name, status, createdAt 等）を記録
   - リレーション（外部キー、参照型）を特定

3. ルーティング・エンドポイントの抽出:
   - routes/, pages/, app/ 配下のファイルからパスを抽出
   - HTTP メソッド + パス + ハンドラの対応
   - API エンドポイントの一覧化

4. 画面構成の抽出（フロントエンドがある場合）:
   - pages/, views/, screens/ 配下のコンポーネント
   - ルーティング定義からページ一覧を生成
   - 主要コンポーネント（フォーム、リスト、詳細、ダッシュボード）の特定

5. ビジネスロジックの所在特定:
   - services/, usecases/, domain/ 配下のファイル
   - 主要な関数・メソッドの目的を推定
   - バリデーションロジックの所在

6. 状態管理の特定（フロントエンドがある場合）:
   - Redux, Zustand, Jotai, Recoil, Context API 等
   - グローバル状態とローカル状態の分離
```

## 出力フォーマット

```markdown
## ドメイン分析結果

### アーキテクチャパターン
- パターン: {Clean / Layered / Flat / ページベース}
- 主要ディレクトリ: {src/ 直下の構成}

### エンティティ一覧
| エンティティ | ファイル | 主要フィールド | リレーション |
|-------------|---------|--------------|-------------|
| {名前} | {パス} | {フィールド列挙} | {参照先} |

### API エンドポイント一覧
| メソッド | パス | ハンドラ | 概要 |
|---------|------|---------|------|
| {GET/POST/...} | {/path} | {関数名} | {推定される目的} |

### 画面構成（フロントエンドがある場合）
| パス | コンポーネント | 種別 | 概要 |
|------|-------------|------|------|
| {/path} | {名前} | {form/list/detail/dashboard} | {推定される目的} |

### ビジネスロジック
| サービス/ユースケース | ファイル | 主要機能 |
|---------------------|---------|---------|
| {名前} | {パス} | {機能概要} |
```
