# 品質ルール

ユーザーストーリーの品質を検証するためのルール定義。
Step 5（品質チェック + 自動補正）で使用する。

## 曖昧語禁止リスト

以下の語句がユーザーストーリーまたは AC に含まれている場合、具体化を求める。

| 禁止語 | 具体化の例 |
|--------|----------|
| 適切に | → 「3 秒以内に」「エラーメッセージを表示して」 |
| など | → 列挙を完成させる。「りんご、みかん、バナナ」 |
| 必要に応じて | → 条件を明示。「IF 在庫が 0 THEN」 |
| 可能であれば | → Must / Should / Could に分類 |
| いくつかの | → 具体的な数値。「3 件以上」 |
| 十分な | → 定量基準。「100 件以上」 |
| 速い / 速く | → 「500ms 以内」「3 秒以内」 |
| 多い / 少ない | → 具体的な閾値 |
| 安全に | → セキュリティ要件を明示。「TLS 1.3 で暗号化」 |
| 使いやすい | → 具体的な UX 基準。「3 ステップ以内で完了」 |
| 高品質な | → 品質基準を明示 |
| 柔軟に | → 設定項目や拡張ポイントを明示 |

## EARS 記法分類基準

### 分類判定ルール

| 要件の特徴 | EARS 分類 |
|-----------|-----------|
| 常に成り立つべき機能 | SHALL |
| 特定のイベント発生時に起動する機能 | WHEN-THEN |
| 特定の条件下でのみ成り立つ機能 | IF-THEN |
| パフォーマンス・セキュリティ等の制約 | WHERE |
| あれば望ましいオプション機能 | MAY |
| 明示的に禁止すべき動作 | MUST NOT |

### 未分類の検出

以下のパターンは EARS 未分類の可能性が高い:
- 「〜できる」だけで条件が不明 → SHALL or IF-THEN に分類
- 「〜してはいけない」→ MUST NOT に分類
- 「〜の場合は」→ IF-THEN or WHEN-THEN に分類
- 数値制約（「N 以内」「N 以上」）→ WHERE に分類

## テスト可能性チェック基準

| チェック項目 | 合格基準 |
|------------|---------|
| Given/When/Then 形式 | 全 AC が Gherkin 形式であること |
| 定量基準 | WHERE 要件に具体的な数値があること（「速い」ではなく「500ms 以内」） |
| 入力の明示 | Given に具体的な前提条件があること |
| 期待結果の明示 | Then に検証可能な結果があること（「正しく表示される」ではなく「ステータス 200 と JSON を返す」） |
| エッジケース | 境界値に関する AC があること（max, min, 0 件, 上限） |

## 完全性チェック基準

| チェック項目 | 合格基準 |
|------------|---------|
| ペルソナ | 最低 1 人定義されていること |
| Epic | 最低 1 つ定義されていること |
| Story/Epic 紐付け | 全 Story が いずれかの Epic に属すること（孤児 Story 禁止） |
| 正常系 AC | 全 Story に正常系 AC が最低 1 件あること |
| 異常系 AC | 全 Story に異常系 AC が最低 1 件あること |
| Non-Goals | Non-Goals が明示されていること（1 項目以上） |
| ID 連番 | P-XXX, Epic-XXX, US-XXX, AC-XXX-Y が連番で欠番がないこと |
| EARS 分類 | 全 Story に EARS 分類が付与されていること |

## 信頼度ルール

### 信頼度レベル判定基準

| レベル | 記号 | 判定基準 |
|--------|------|---------|
| Blue | 🔵 | ユーザーがインタビューで直接述べた内容。引用可能 |
| Yellow | 🟡 | ユーザーの回答から合理的に推論できる内容。例: 「EC サイト」→ 「商品一覧が必要」 |
| Red | 🔴 | ユーザーが言及していないが、AI が一般的な知識から補完した内容 |

### 信頼度の警告ルール

| 条件 | アクション |
|------|----------|
| Red が全体の 30% 以上 | 追加質問を提案する（強制しない）。「いくつかの要件が推測に基づいています。追加でヒアリングしますか？」 |
| Red が全体の 50% 以上 | インタビュー不足として強く追加質問を推奨。「要件の半数以上が推測です。追加ヒアリングを強く推奨します」 |
| Must の Story に Red の AC がある | 個別に警告。「MVP 必須機能の受け入れ基準が推測に基づいています。確認してください」 |

### 信頼度の昇格

- ユーザーが Red の内容を確認 → Yellow に昇格
- ユーザーが Yellow の内容を明示的に承認 → Blue に昇格
- 品質チェックで自動補完した内容 → 常に Red
