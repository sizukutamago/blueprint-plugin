# インタビュー質問テンプレート

Double Diamond パターンに基づく構造化質問。
各質問は AskUserQuestion ツールで実行する（テキスト質問禁止）。

## Diamond 1: Discover（発散フェーズ）

### Q1: プロジェクト概要

- **question**:
  - README/PRD がある場合: `"README を読みました。{要約を 1-2 文で提示}。この理解で合っていますか？ 補足があれば教えてください"`
  - brownfield の場合: `"既存コードを分析しました。{context_summary の要約}。今回追加・変更したい機能を教えてください"`
  - 何もない場合: `"何を作りたいですか？ 1-2文で教えてください"`
- **header**: `"プロジェクト概要"`
- **type**: free_text
- **EARS 分類**: SHALL（主要機能の特定）
- **目的**: ビジネスゴールと主要機能の把握
- **フォローアップ例**: 「それは誰のどんな課題を解決しますか？」
- **適応ルール**: README/PRD/context_summary がある場合は AI が先に要約を提示して確認を取る方が効率的。ゼロから説明させない

### Q2: ターゲットユーザー

- **question**: `"主なユーザーは誰ですか？"`
- **header**: `"ターゲットユーザー"`
- **options**:
  - label: `"一般消費者（B2C）"` / description: `"エンドユーザーが直接利用する"`
  - label: `"企業担当者（B2B）"` / description: `"業務利用。管理画面やダッシュボードが中心"`
  - label: `"社内ユーザー（社内ツール）"` / description: `"自社メンバーが利用する業務ツール"`
  - label: `"開発者（API/SDK/CLI）"` / description: `"開発者向けのツールやサービス"`
  - label: `"複数タイプ"` / description: `"上記の組み合わせ（詳細を教えてください）"`
- **フォローアップ**: 「そのユーザーが最も困っていることは何ですか？」
- **目的**: ペルソナの特定（P-XXX）

### Q3: 主要ユースケース

- **question**: `"ユーザーがこのシステムで行う主な操作を 3 つ挙げてください"`
- **header**: `"主要ユースケース"`
- **type**: free_text
- **フォローアップ**: 「それぞれの操作の頻度は？（日次 / 週次 / 月次 / 不定期）」
- **EARS 分類**: SHALL（各ユースケースを SHALL 要件に変換）
- **目的**: Epic の種を収集（Epic-XXX）

### Q4: プラットフォーム

- **question**: `"どのプラットフォームで動作しますか？"`
- **header**: `"プラットフォーム"`
- **options**:
  - label: `"Web アプリケーション"` / description: `"ブラウザで動作する Web アプリ"`
  - label: `"モバイルアプリ（iOS/Android）"` / description: `"ネイティブまたはクロスプラットフォーム"`
  - label: `"CLI ツール"` / description: `"コマンドラインで動作するツール"`
  - label: `"API サーバー（バックエンドのみ）"` / description: `"API を提供するサーバー"`
  - label: `"バッチ処理 / ジョブ"` / description: `"定期実行やデータ処理"`
  - label: `"デスクトップアプリ"` / description: `"Electron 等のデスクトップアプリケーション"`
  - label: `"その他"` / description: `"上記に当てはまらない場合"`
- **目的**: スコープとアーキテクチャの方向性

## Diamond 1: Define（収束フェーズ）

### Q5: MVP 優先度

- **header**: `"MVP スコープ"`
- **type**: free_text（Q3 の回答を元に MoSCoW 分類を提示）
- **question**: Q3 の回答を元に動的生成する。例:
  ```
  "Q3 で挙がった機能を優先度で分類してください。
   1. {機能A}
   2. {機能B}
   3. {機能C}
   それぞれ Must（必須）/ Should（推奨）/ Could（可能なら）のどれですか？"
  ```
- **目的**: Must / Should / Could の分類

### Q6: 非目標

- **question**: `"今回のスコープに含めないものは何ですか？"`
- **header**: `"Non-Goals"`
- **type**: free_text
- **補足**: 「明示的に『やらないこと』を決めておくと、後で判断に迷いません」
- **目的**: スコープの明確な境界設定
- **フォローアップ例**: 「多言語対応は今回スコープに入りますか？」

## Diamond 2: Develop（深掘りフェーズ）

### Q7: ビジネスルール・制約

- **question**: `"このシステムに特有のビジネスルールや制約はありますか？"`
- **header**: `"ビジネスルール"`
- **type**: free_text
- **補足**: 「例: 金額計算の端数処理、ステータス遷移のルール、権限による制限 等」
- **フォローアップ**: 回答に応じた深掘り質問（具体的な値・条件・例外を引き出す）
- **EARS 分類**: IF-THEN / WHEN-THEN / MUST NOT

### Q8: 成功指標

- **question**: `"このプロジェクトの成功をどう測定しますか？"`
- **header**: `"成功指標"`
- **type**: free_text
- **補足**: 「例: DAU、コンバージョン率、作業時間の削減、エラー率 等」
- **EARS 分類**: WHERE（定量基準 → NFR の根拠）
- **目的**: KPI / SLO の種を収集

## Diamond 2: Deliver（確認フェーズ）

### Q9: 外部連携・データソース（条件付き）

- **condition**: Q7 の回答で外部サービスへの言及がある場合、または Q4 で API サーバーを選択した場合
- **question**: `"連携する外部サービスやデータソースはありますか？"`
- **header**: `"外部連携"`
- **options**:
  - label: `"決済（Stripe / PayPay 等）"` / description: `"オンライン決済サービス"`
  - label: `"認証（Auth0 / Firebase Auth 等）"` / description: `"外部認証プロバイダー"`
  - label: `"メール / 通知（SendGrid / FCM 等）"` / description: `"メール送信やプッシュ通知"`
  - label: `"AI / LLM（OpenAI / Claude 等）"` / description: `"AI モデルの API 呼び出し"`
  - label: `"その他"` / description: `"上記以外の外部サービス"`
  - label: `"なし"` / description: `"外部連携は不要"`
- **目的**: external Contract の種を収集

### Q10: NFR・リスク（条件付き）

- **condition**: Q8 の回答で性能・セキュリティ・可用性への言及がある場合
- **question**: `"パフォーマンス、セキュリティ、可用性で特に重要な要件はありますか？"`
- **header**: `"非機能要件"`
- **type**: free_text
- **補足**: 「例: ページ読み込み 3 秒以内、個人情報の暗号化、99.9% 稼働率 等」
- **EARS 分類**: WHERE
- **目的**: NFR-{CAT}-XXX の導出

## 質問の適応ルール

- Q3 の回答数が少ない（1-2 個）場合、フォローアップで追加ユースケースを引き出す
- Q4 の回答が「その他」の場合、フォローアップで具体的なプラットフォームを確認
- Q7 の回答が薄い場合、Q3 の各ユースケースについて「異常系は？」「制限は？」と深掘り
- Q8 の回答が「特にない」の場合、Q9/Q10 はスキップ
- brownfield の場合、context_summary の内容を質問に反映（「既存の X 機能を拡張しますか？」等）
