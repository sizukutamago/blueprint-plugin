# 設計書フォーマット基準

`/generate-docs` が出力する設計書の必須セクション・品質基準を定義する。
各設計書は `docs/` 配下に生成され、`core/output-structure.md` のディレクトリ構造に従う。

## 03_architecture/

### architecture.md

| セクション | 内容 |
|-----------|------|
| 技術スタック選定理由 | 各技術の選定理由・バージョン・代替候補 |
| アーキテクチャパターン | 採用パターン（MVC, Clean Architecture 等） |
| レイヤー構成 | 各レイヤーの責務と依存方向 |
| システム境界定義 | 外部システムとの境界・通信方式 |
| NFR 測定可能性定義 | target / measurement / pass_criteria の 3 要素テーブル |

### adr.md

各 ADR は以下の構成:

| セクション | 内容 |
|-----------|------|
| コンテキスト | 意思決定が必要になった背景 |
| 決定 | 採用した方針 |
| 理由 | 決定の根拠 |
| 代替案 | テーブル形式で候補と比較 |
| 影響 | 決定による影響範囲 |

### security.md

| セクション | 内容 |
|-----------|------|
| 認証実装詳細 | JWT 署名方式、トークン保存、セッション管理 |
| 脆弱性対策 | OWASP Top 10 対応テーブル |
| データガバナンス | データ分類、PII 対応、ライフサイクル |
| PII マスキングルール | マスキング対象と方式 |

### infrastructure.md

| セクション | 内容 |
|-----------|------|
| 可用性目標 | 稼働率、RTO、RPO |
| 構成図 | Mermaid グラフ形式 |
| スケーリング方針 | 水平/垂直スケーリング戦略 |
| ネットワーク構成 | パブリック/プライベート分離 |

### cache_strategy.md

| セクション | 内容 |
|-----------|------|
| レイヤー別設計 | ブラウザ、CDN、アプリケーション、DB |
| キャッシュキー設計 | キー命名規則と構造 |
| 無効化戦略 | TTL、イベント駆動、手動 |
| TTL 設計 | リソース別 TTL 値 |

## 04_data_structure/

### data_structure.md

| セクション | 内容 |
|-----------|------|
| 命名規則 | テーブル名、カラム名の命名規約 |
| エンティティ定義 | TypeScript インターフェース + フィールド詳細テーブル + データ分類 |
| フロントエンド専用型 | Form 型、State 型の定義 |
| バリデーションルール | 各フィールドの制約条件 |
| 物理データベース設計 | テーブル定義、インデックス、制約、容量見積もり |
| データ暗号化方式 | 暗号化対象と方式 |
| マイグレーション戦略 | スキーマ変更管理方式 |

## 05_api_design/

### api_design.md

| セクション | 内容 |
|-----------|------|
| 設計方針 | 設計原則、バージョニング、ベース URL |
| 命名規則 | RESTful URL パターン |
| 認証・認可 | 認証方式と権限モデル |
| API 一覧 | API-ID、メソッド、エンドポイント、概要、認証要否のテーブル |
| API 詳細 | 各 API のパラメータ、リクエスト/レスポンス、エラー |
| 共通仕様 | 日時形式、ページネーション、レート制限 |
| エラーコード一覧 | RFC 7807 形式のエラーレスポンス定義 |

### integration.md

外部連携がない場合も「連携なし」を明記すること。

| セクション | 内容 |
|-----------|------|
| 連携システム一覧 | 外部システムと連携方式 |
| 連携方式 | 同期 API、非同期、Webhook、バッチ |
| 連携先詳細 | エンドポイント、認証、タイムアウト、リトライ |
| 障害時対応 | フォールバック戦略 |

| エラーハンドリング | エラー種別と UI 表示 |

## 横断ルール

| ルール | 説明 |
|--------|------|
| ID 体系 | FR, NFR, SC, API, ENT, ADR, CON（`core/id-system.md` 参照） |
| テーブル形式 | 定義は原則テーブルで記載 |
| Mermaid 図 | アーキテクチャはグラフ、画面遷移は stateDiagram-v2 |
| トレーサビリティ | FR/NFR → 設計成果物 → テストケースのマッピングを維持 |
| 条件付き生成 | backup_restore_dr.md, migration_plan.md はプロジェクト要件による |
| 確信度マーカー | 各セクションに `<!-- confidence: high/medium/low -->` を付与 |
